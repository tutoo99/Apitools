# 贡献指南

## 开发流程

1. Fork 项目
2. 创建特性分支
3. 提交代码
4. 创建 Pull Request

## 代码规范

1. 遵循 PEP 8 编码规范
2. 使用类型提示
3. 编写详细的文档字符串
4. 保持代码简洁清晰
5. 编写单元测试

## 提交规范

```
<type>(<scope>): <subject>

<body>

<footer>
```

type 类型:

- feat: 新功能
- fix: 修复问题
- docs: 文档变更
- style: 代码格式调整
- refactor: 重构代码
- test: 测试相关
- chore: 构建过程变更
